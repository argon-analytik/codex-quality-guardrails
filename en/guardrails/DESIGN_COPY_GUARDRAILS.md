# COPY GUARDRAILS

## Visible English Text
For all visible English-language text in UI, emails, PDFs, portals, notifications, assistant responses, and any other communication, the following applies:
- use typographic quotation marks `“ ”` and apostrophes `’` in visible prose where typographically appropriate
- use quotation and punctuation style consistently instead of mixing styles arbitrarily
- use em dashes `—` sparingly for clear parenthetical breaks; by default, do not surround them with spaces unless your product style guide requires it
- use en dashes `–` for ranges where appropriate
- use the ellipsis character `…` in polished visible copy where typography matters, instead of three separate dots
- keep casing, punctuation, and label style consistent across similar UI elements

## Exceptions
ASCII-friendly or plain-text forms are allowed in:
- code
- identifiers
- slugs
- URLs
- filenames
- paths
- CLI commands
- technical keys
- `kebab-case` / `snake_case`

## Rule
Separate technical strings clearly from human language.
Human-facing language should read like clean, correct, locale-aware English typography.

These are general English defaults.
If the product follows a specific editorial standard such as US, UK, Australian, or another national style, adapt the rules consistently for that locale.
